1. Gauss elimination method.

Sure, the Gauss elimination method is a systematic way to solve systems of linear equations. The idea is to convert the system's augmented matrix into an upper triangular form, which simplifies the process of solving for the variables through back-substitution. For example, let's consider this system of equations:
2x + 3y = 8
x - y = 2 We can use Gauss elimination to eliminate one variable and then solve for the other, making the process straightforward.

2. Bubble sort.

Bubble sort is one of the simplest sorting algorithms. It works by repeatedly stepping through the list, comparing adjacent elements, and swapping them if they are in the wrong order. This way, smaller values "bubble" up to the top of the list. For instance, if we have the list [5, 2, 9, 1], we compare 5 and 2. Since 5 is greater than 2, we swap them. We continue this process, and after several iterations, the list becomes sorted as [1, 2, 5, 9].

3. Euclidean algorithm.

The Euclidean algorithm is a method to find the greatest common divisor (GCD) of two integers. It relies on the principle that the GCD of two numbers also divides their difference. For example, to find the GCD of 48 and 18:

  • We divide 48 by 18, which gives a remainder of 12.
  • Then, we divide 18 by 12, resulting in a remainder of 6.
  • Next, we divide 12 by 6, and the remainder is 0.
The last non-zero remainder, 6, is the GCD of 48 and 18.

4. Eratosthenes sieve.

The Sieve of Eratosthenes is an ancient algorithm for finding all prime numbers up to a certain limit. It systematically eliminates the multiples of primes, leaving only the primes themselves. For example, to find all primes less than 10:

  1. List numbers from 2 to 9.
  2. Starting with 2, cross out all multiples of 2.
  3. Move to the next uncrossed number, 3, and cross out its multiples.
  4. The remaining uncrossed numbers are primes: 2, 3, 5, and 7.

5. Newton methods.

Newton's method, or the Newton-Raphson method, is an iterative technique to approximate the roots of a real-valued function. It uses the function's derivative to converge to a root. For example, if we want to find the square root of a number a, we can set up the function f(x) = x² - a (read as "f of x equals x squared minus a"). Starting with an initial guess x0 (read as "x sub zero"), we apply:
xn+1 = xn - f(xn) / f′(xn) (read as "x sub n plus one equals x sub n minus f of x sub n divided by f prime of x sub n")
This process quickly converges to the actual square root of a.

6. Horner method.

The Horner's method is an efficient algorithm for evaluating polynomials at a given point and for polynomial division. It restructures the polynomial to minimize the number of multiplications. For instance, to evaluate P(x) = 2x³ - 6x² + 2x - 1 at x = 3, we rewrite it using Horner's method:
P(x) = ((2x - 6)x + 2)x - 1 (read as "P of x equals open parenthesis open parenthesis two x minus six close parenthesis times x plus two close parenthesis times x minus one")
By evaluating this nested expression step by step, we find that P(3) = 5.

7. Properties of normal distribution.

The normal distribution is a continuous probability distribution that is symmetrical around the mean, depicting that data near the mean are more frequent in occurrence. It's often referred to as the bell curve due to its shape. Key properties include:

  • It's defined by its mean (μ) and standard deviation (σ) (read as "mu" and "sigma").
  • Approximately 68% of the data falls within one standard deviation of the mean.
  • The curve is symmetrical, so the mean, median, and mode are all equal.
For example, in human height, most people are of average height, with fewer being extremely tall or short, which fits the normal distribution.

8. How to calculate Fibonacci number.

Calculating a Fibonacci number involves summing the two preceding numbers in the sequence, starting from 0 and 1. There are several methods to compute it, including recursion, iteration, and using formulas. For example, using iteration:

  1. Start with F0 = 0 and F1 = 1 (read as "F sub zero equals zero" and "F sub one equals one").
  2. Then Fn = Fn-1 + Fn-2 (read as "F sub n equals F sub n minus one plus F sub n minus two").
  3. So, F2 = 1, F3 = 2, F4 = 3, F5 = 5, and so on (read as "F sub two equals one", "F sub three equals two", "F sub four equals three", "F sub five equals five").
Alternatively, for larger n, we can use Binet's formula, which involves the golden ratio, to compute Fn directly.

9. Object oriented programming.

Object-oriented programming (OOP) is a programming paradigm that uses "objects" to design applications and computer programs. It emphasizes concepts like classes, inheritance, and polymorphism. Key principles include:

  • Encapsulation: Bundling data with methods that operate on that data.
  • Inheritance: Creating new classes from existing ones.
  • Polymorphism: Methods can perform differently based on the object.
  • Abstraction: Hiding complex reality while exposing only the necessary parts.
For example, if we're programming a game, we might have a base class Character and derive classes like Hero and Villain, each with their own specific properties and methods.

10. Relational model of the database. Tables, primary and foreign keys, database table relationships.

The relational model organizes data into tables (relations) composed of rows and columns. It uses keys to establish relationships between tables. - A primary key uniquely identifies each record in a table. - A foreign key is a field in one table that refers to the primary key of another table, establishing a link between them. For example, in a database, we might have a Customers table with a CustomerID as the primary key, and an Orders table where each order record includes a CustomerID foreign key. This relationship allows us to associate orders with the customers who made them.
